# Privacy Policy Draft

Status: draft for a blocked internal-testing candidate.

Fishing Maxxed currently stores only local UI preference state in the present build. The blocked release candidate does not transmit catch photos, measurements, locations, accounts, analytics, or cloud backups to a Maxxed-operated service.

Current on-device behavior:

- Theme selection is stored in app-private local storage.
- No sign-in flow is present.
- No network synchronization or remote API dependency is implemented.
- No third-party ad SDKs are present.

Important limitation:

- The intended production app brief requires catch capture, species confirmation, private exact location handling, journal management, and a California rule engine. Those capabilities are not implemented in the current build, so this draft must be revised before any real Play upload if those features are added.
